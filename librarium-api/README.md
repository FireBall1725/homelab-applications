# librarium-api

> Go backend for the self-hosted Librarium library tracker

## Overview

Librarium is a self-hosted library tracker (books, ebooks, audiobooks) built around an abstract work / concrete edition (FRBR) data model. This chart deploys the Go API backend and its CloudNative-PG managed PostgreSQL cluster. A companion `librarium-web` chart (React frontend) is planned.

## Chart Details

| | |
|---|---|
| **Upstream Project** | [librarium](https://github.com/fireball1725/librarium) |
| **Upstream Chart** | `librarium-api` (vendored from `librarium/api/deploy/helm/librarium-api`) |
| **Chart Version** | `0.1.0` |
| **App Version** | `26.4.0` |
| **Common Library** | `common` 5.0.3 ([firelabs-helm-common](https://fireball1725.github.io/firelabs-helm-common/)) |

This is Option A (self-contained): the upstream chart is copied verbatim into `templates/` and `charts/`. Only `values.yaml` contains homelab overrides, plus a `SealedSecret` template for `JWT_SECRET`. A future upstream refresh is a clean re-copy + reapply-overrides.

## Ingress

| Host | Description |
|------|-------------|
| `librarium.k8s.firekatt.ca` | Librarium API (internal LAN only) |

The `*.k8s.firekatt.ca` zone resolves inside the LAN only, so `REGISTRATION_ENABLED=true` is not external exposure.

## Persistence

| Volume | Size | Storage Class | Notes |
|--------|------|---------------|-------|
| `librarium-api-covers` | 10Gi | `longhorn` | Cover art (small, RWO) |
| `librarium-db` (CNPG) | 10Gi | `longhorn` | PostgreSQL cluster, PG 17.2-3 |

`persistence.media` is disabled for now (ebook/audiobook blobs are not hosted by this instance yet). Flip it back on and decide RWX vs RWO + NFS `existingClaim` when that use case lands.

## Secrets

Uses [Sealed Secrets](../SECRETS.md) for sensitive values.

| Secret | Contents | Source |
|--------|----------|--------|
| `librarium-api-jwt` | `JWT_SECRET` | SealedSecret (`templates/sealed-secret-jwt.yaml`) |
| `librarium-db-app` | PostgreSQL DSN (`uri` key) | Auto-generated by CNPG operator |

## Notes

- The upstream chart, when `.Values.secret` is populated, auto-renders a Secret named `<fullname>` and attaches it via `envFrom`. We intentionally disable that (`secret: {}`) and instead inject `JWT_SECRET` via `env.JWT_SECRET.valueFrom.secretKeyRef` pointing at the SealedSecret-managed `librarium-api-jwt`. This keeps secret rotation decoupled from chart values.
- `DATABASE_URL` is pulled from the CNPG-generated `librarium-db-app` Secret (`uri` key).
- PostgreSQL is provisioned via a `Cluster` CR managed by [CloudNative-PG](../cloudnative-pg/).
